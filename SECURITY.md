# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in **RAID Calculator**, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, please send a detailed report to:

- **Email:** [hi@alirashidnahal.com](mailto:hi@alirashidnahal.com)
- **GitHub:** Open a [private security advisory](https://github.com/alirashidnahal/raid_calculator/security/advisories/new)

### What to Include

When reporting a vulnerability, please include as much of the following as possible:

- A clear description of the vulnerability
- Steps to reproduce the issue
- The potential impact of the vulnerability
- Any suggested fixes or patches (if available)

### Response Timeline

- **Acknowledgment:** Within **48 hours** of receiving the report
- **Initial Assessment:** Within **7 days**
- **Resolution:** Depending on severity, typically within **30 days**

### What to Expect

- You will receive an acknowledgment confirming receipt of your report.
- We will investigate and validate the reported vulnerability.
- If the vulnerability is accepted, we will work on a fix and coordinate disclosure.
- If the vulnerability is declined, we will provide an explanation.
- Credit will be given to the reporter (unless anonymity is requested).

## Security Best Practices

This project is a client-side web application. All RAID calculations are performed locally in the user's browser. No data is sent to any server.

### Dependencies

We regularly monitor dependencies for known vulnerabilities. If you notice an outdated or vulnerable dependency, please let us know via the methods above.

## Disclosure Policy

We follow a coordinated disclosure process. Please allow us reasonable time to address any reported vulnerability before making it public.

Thank you for helping keep RAID Calculator safe!
